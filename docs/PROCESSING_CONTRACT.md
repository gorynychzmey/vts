# Processing Contract Audit

Audit date: 2026-02-27

This document verifies the current implementation against the detailed processing contract (download/transcribe/summarize behavior).

Status legend:
- `PASS`: implemented as described
- `PARTIAL`: implemented with important differences
- `MISSING`: not implemented

## Option Names

The detailed contract uses:
- `audio_only`
- `do_transcribe`
- `do_summary`
- `language`

Current API request model uses:
- `audio_only`
- `transcript` (equivalent to `do_transcribe`)
- `summary` (equivalent to `do_summary`)
- `language`

Validation is enforced: `summary=true` requires `transcript=true`.

## 1) Task Scenarios

Status: `PASS`

Implemented combinations:
- `audio_only=false`, `transcript=false`, `summary=false` -> download only
- `audio_only=false`, `transcript=true`, `summary=false` -> download + transcription
- `audio_only=false`, `transcript=true`, `summary=true` -> download + transcription + summarization
- `audio_only=true`, `transcript=true`, `summary=true` -> audio download + transcription + summarization
- `audio_only=true`, `transcript=false`, `summary=true` -> rejected by validation

## 2) Download Video/Audio

Status: `PASS`

Implemented:
- `yt-dlp` Python API with progress hooks.
- Separate `video_progress` and `audio_progress` SSE events.
- `audio_only=true` skips video download.
- `audio_only=false` stores merged `media/video.mkv`.
- Original audio is extracted with `ffmpeg -c copy` into `media/audio.original.<ext>`.
- Download phase SSE includes `video`, `audio`, `merge`, `postprocess`.

## 3) Audio Segmentation

Status: `PASS`

Implemented:
- Target length 300s, search window +/-30s, overlap 3s.
- Silence detection uses `-30dB` with `d=1.0`.
- Output segments are WAV 16k mono.
- Segment files are named `segments/0001.wav`, `segments/0002.wav`, ...
- Segment timestamps are persisted in DB right after segmentation.

## 4) Transcription

Status: `PASS`

Implemented:
- Runs only when `transcript=true`.
- Per-task concurrency is bounded by contract limit (`<=2`), global heavy slot is enforced.
- Global heavy slot limit applies.
- Whisper `/asr` call sends `language` and `initial_prompt` (tail from previous segment).
- Stores normalized segments and words in DB.
- Stores raw artifact file `asr/segments_raw.json`.

## 5) Transcript Merge

Status: `PASS`

Implemented:
- Timestamp-based deduplication (no fuzzy matching).
- Plain UTF-8 transcript file is produced (`outputs/transcript.txt`).
- Merge rule follows previous segment end cutoff (`t_start < prev_segment_end` is dropped).

## 6) Summarization

Status: `PASS`

Implemented:
- Runs only when `summary=true` and `transcript=true`.
- llama.cpp OpenAI-compatible endpoint.
- Token sliding window 2000 with 15% overlap.
- Prompts loaded from `/opt/vts/prompts`.
- Window outputs are stored in `summary/window_01.txt`, `summary/window_02.txt`, ...
- Final output is stored in `summary/final.md` (plus JSON companion).

## 7) Retention and Cleanup

Status: `PASS`

Implemented:
- `segments/*.wav` removed after transcript merge.
- Media files in `media/` cleaned by TTL; TTL can be disabled (`media_ttl_hours <= 0`).
- Transcript and summary outputs are retained.
- Raw ASR JSON artifact `asr/segments_raw.json` is retained.

## 8) Per-task Logging

Status: `PASS`

Implemented:
- Per-task log file `logs/task.log`.
- yt-dlp and ffmpeg output is written to the task log.
- ASR and LLM responses are logged with safe truncation.

## 9) Recovery After Failure

Status: `PASS`

Implemented:
- Step idempotency through output checks and DB step statuses.
- Worker startup recovery:
  - stale `running` tasks are requeued,
  - queued tasks are restored to Redis queue.
- Completed steps are not repeated when outputs are present.

## 10) Scenario Matrix

Status: `PASS` for control-flow, see sections above for artifact-format differences.

| audio_only | do_transcribe | do_summary | Current behavior |
| --- | --- | --- | --- |
| false | false | false | Download only |

| false | true | true | Download -> transcribe -> summarize |

## 11) Responsibility Boundaries

Status: `PASS`

Implemented:
- No mp3 artifacts are generated by current pipeline.
- No fuzzy transcript merge.
- Deployment is manual.
